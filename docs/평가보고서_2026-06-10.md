## 2. 보안 평가

### 2-1. 데이터베이스 보안

**Row Level Security (RLS)** 를 모든 테이블에 적용하였다.

```
posts 테이블
  ├── SELECT: 모든 사용자 허용 (공개 게시판)
  ├── INSERT: 인증된 사용자만 (auth.uid() = user_id)
  ├── UPDATE: 본인 작성글만 (auth.uid() = user_id)
  └── DELETE: 본인 작성글만 (auth.uid() = user_id)

comments 테이블
  ├── SELECT: 모든 사용자 허용
  ├── INSERT: 인증된 사용자만
  └── DELETE: 본인 댓글만
```

**평가**: 서버 측에서 권한을 검증하므로 클라이언트에서 임의 조작이 불가하다. SQL Injection은 Supabase ORM이 방어한다.

### 2-2. 인증 보안

- 비밀번호는 Supabase가 bcrypt로 해시 처리 (직접 저장하지 않음)
- JWT 토큰 만료: 1시간 (자동 갱신)
- 환경변수(`.env`)는 `.gitignore`에 등록 → GitHub에 노출되지 않음
- Anon Key는 공개용으로 설계된 키 (클라이언트 노출 허용)

### 2-3. 보완 필요 사항

| 항목 | 위험도 | 권장 조치 |
|------|--------|-----------|
| 공지사항 작성 권한 | 중 | 관리자 role 분리 필요 |
| 글자 수 제한 미적용 | 하 | content 최대 길이 제한 추가 |
| XSS 방어 | 하 | `whitespace-pre-wrap` 렌더링 사용 중 (HTML 미실행) — 현재 안전 |
